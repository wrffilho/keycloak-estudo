# ADR-004: Integracao Backend Keycloak Via Admin REST API

## Status

Accepted

## Date

2026-05-11

## Context

O backend precisa consultar o Keycloak para obter o catalogo de menus do client `laboratorio-menu`. A integracao deve ficar no backend, nunca no frontend, porque credenciais administrativas ou de servico nao podem ser expostas no navegador.

O objetivo da V1 e ler resources/attributes do client de menu e montar um JSON limpo para o Vue.

## Decision

A V1 usara a Admin REST API do Keycloak para consultar o client `laboratorio-menu`, ler resources/attributes de menu e obter informacoes necessarias para montar a arvore permitida para o usuario logado.

O backend usara credencial de servico com menor privilegio possivel e isolara essa integracao em um servico dedicado.

## Alternatives Considered

### Alternative 1: Protection API

- **Description**: Usar a Protection API do Keycloak para gerenciar/consultar recursos do resource server.
- **Pros**: Apropriada para resource servers que gerenciam recursos remotamente.
- **Cons**: Menos direta para a V1, que precisa ler um catalogo configurado no realm.
- **Why rejected**: A Admin REST API e mais simples para consultar configuracao administrativa do client de menu.

### Alternative 2: Dados locais no backend

- **Description**: Manter catalogo de menu em arquivo ou codigo local.
- **Pros**: Mais simples de implementar e testar.
- **Cons**: Nao valida a hipotese central de menu administrado no Keycloak.
- **Why rejected**: O aprendizado desejado e consultar o catalogo real no Keycloak.

## Consequences

### Positive

- Mantem o frontend desacoplado do Keycloak.
- Permite ler metadados ricos de resources/attributes.
- Centraliza a traducao Keycloak -> JSON de menu no backend.

### Negative

- Exige configuracao de credencial de servico.
- Aumenta dependencia operacional do backend em relacao ao Keycloak.
- Requer tratamento cuidadoso de erros quando o Keycloak estiver indisponivel.

### Risks

- **Privilegio excessivo da credencial**: mitigar usando menor privilegio possivel.
- **Acoplamento a endpoints administrativos**: mitigar isolando chamadas em uma classe/servico especifico.

## Implementation Notes

- O TechSpec deve prever configuracoes de URL, realm, client de menu e credencial de servico.
- A resposta do backend deve esconder detalhes administrativos do Keycloak.
